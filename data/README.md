# Data Directory

This directory contains data files generated by the Python and C++ tools.

## Contents

### Ticker Data (Time Series)
- `kraken_ticker_history.csv` - Historical ticker updates from WebSocket v1
- `kraken_ticker_history_v2.csv` - Historical ticker updates from WebSocket v2

### Volume Analysis
- `kraken_top500_usd_volume.csv` - Top 500 trading pairs by 24h USD volume
- `kraken_usd_volume.csv` - USD volume data for all pairs
- `kraken_pairs_liquidity.csv` - Liquidity metrics for trading pairs
- `missing_conversions.json` - Quote currencies missing USD conversion

### Reference Data
- `kraken_tickers.txt` - List of available trading pairs

## Git Ignore

Data files are typically excluded from version control (.gitignore) because:
- They can be large (ticker history files can grow to MBs)
- They change frequently
- They can be regenerated from the scripts
- They may contain sensitive or time-specific data

## Regenerating Data

All data files can be regenerated using the Python scripts in `../python/`:

```bash
# Volume analysis
python python/rest_api/get_volume_all_pairs.py
python python/rest_api/find_missing_conversions.py

# Live ticker data
python python/websocket/query_live_data_v2.py

# List all pairs
python python/rest_api/list_all_pairs.py
```

Or using C++ tools in `../cpp/build/`:

```bash
# Level 1 ticker data
./cpp/build/retrieve_kraken_live_data_level1 -p "BTC/USD,ETH/USD" -o data/ticker.csv

# Level 2 order book data (raw)
./cpp/build/retrieve_kraken_live_data_level2 -p "BTC/USD" -o data/raw_book.jsonl

# Level 2 processed snapshots
./cpp/build/process_orderbook_snapshots -i data/raw_book.jsonl --interval 1s -o data/snapshots.csv

# Level 3 order-level data (requires authentication)
export KRAKEN_WS_TOKEN="your_token"
./cpp/build/retrieve_kraken_live_data_level3 -p "BTC/USD" -o data/level3_raw.jsonl
./cpp/build/process_level3_snapshots -i data/level3_raw.jsonl --interval 1s -o data/level3_snapshots.csv
```

## File Formats

### CSV Files
- Comma-separated values
- First row contains column headers
- Compatible with pandas, R, Excel, etc.

### JSON Lines (.jsonl)
- One JSON object per line
- Used for streaming data capture
- Efficient for large datasets

### JSON
- Standard JSON format
- Used for configuration and small datasets
